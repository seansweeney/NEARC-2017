The new ArcGIS API for Python has greatly simplified automation with ArcGIS Online and Portal for ArcGIS. It abstracts away most of the gory low-level details that used to be required to interact with the ArcGIS REST API. ABSTRACT: The ArcGIS API for Python is a Python library for working with maps and geospatial data, powered by web GIS. It provides simple and efficient tools for sophisticated vector and raster analysis, geocoding, map making, routing and directions, as well as for organizing and managing a GIS with users, groups and information items. Get a quick introduction to this powerful API in this session.

What is the Python API?

"Python API for your Web GIS." - Rohit Singh, Lead Developer of Python API
  • Like ArcPy but for AGOL (or Enterprise)
  • Wrapper around AGOL REST API
  • Conda package containing several modules:
    • GIS - Administration, user and group management, content management, etc.
    • Green - Spatial functions and geographic datasets.
    • Blue - Workflow.
    • Orange - Visualization.

What isn't the Python API?

The ArcGIS API for Python is not a replacement for ArcPy. The former is for Web GIS and the latter is for Desktop.

History

2013 Spring NEARC - Managing ArcGIS Server 10.1 with Python

agsextras.py

In 2013, I wrote my own REST API wrappers to use with ArcGIS Server. I later used these for ArcGIS Online as well. It took over 100 lines of Python code to build only the most basic wrapper functions.

2014 NEARC - One Metadata to Rule Them All

In 2014, I attempted to use these wrappers to automate updates of my data and metadata on ArcGIS Online.
One thing I never figured out is how to upload thumbnails.
Portalpy

Bundled with Enterprise < 10.5.1

 ArcREST

Esri had two separate and slightly different open source efforts to try to abstract the REST API for AGOL and Portal.

From Release of the Deprecated Features Plan for ArcGIS 10.5.1

ArcGIS Enterprise 10.5.1 will stop bundling the portalpy module in favor of the ArcGIS API for Python. No further development is planned for this module.

Today I'll be focusing on the GIS Module, which contains five main classes:
  • GIS : represents the GIS, either ArcGIS Online or an ArcGIS Portal
  • User : represents a GIS user
  • Role : represents the role of a GIS user
  • Group : represents a group in the GIS
  • Item : represents an Item in the GIS
